Build your own CRM with AI, step by step

Before you start

Why build instead of rent

Most people pay $100 to $500 a month for a CRM like HubSpot, GoHighLevel, or Salesforce, and after years of paying, they own nothing. The truth is, the core of a CRM is simple: a place to store people, track deals, and see what's happening. With AI doing the heavy lifting, you can build one yourself in an afternoon and own it forever.

I'm not a classically trained developer. I built this whole thing with AI, and this guide hands you the exact path. Whether you want to run your own business on it or build CRMs for other people, this is your starting point.

How to use the prompts. Throughout this guide you'll see orange AI PROMPT boxes. Paste them into your AI coding assistant (Claude, Cursor, or similar) alongside the step, and it will generate the code for you. You describe what you want, the AI builds it.

Step 1

Set up your tools

You need four free things. Don't overthink this, just get them installed.

  • Node.js — the engine that runs the project. Download the LTS version from nodejs.org.
  • A code editor with AI — Cursor or VS Code. This is where you paste the prompts and the AI writes the code.
  • A Supabase account — your free database and login system. Sign up at supabase.com.
  • An AI assistant — Claude or the AI built into Cursor. This is your build partner.

That's the entire stack: Node + React + Supabase, deployed on Vercel. Nothing exotic, nothing expensive.

Step 2

Create the project

Open your terminal and scaffold a fresh React app with Vite. This gives you a clean, fast foundation in seconds.

terminal
# create the project
npm create vite@latest crm -- --template react

# move into it and install
cd crm
npm install

# start it up
npm run dev

Open the local URL it prints (usually http://localhost:5173) and you'll see the starter page. That's your blank canvas.

Step 3

Install the three libraries you need

A CRM needs three things on top of React: pages that route, a database client, and icons. That's it.

terminal
npm install react-router-dom @supabase/supabase-js lucide-react
  • react-router-dom — moves between pages (Dashboard, Leads, Settings).
  • @supabase/supabase-js — talks to your database and handles login.
  • lucide-react — clean icons for the sidebar and cards.

Step 4

Connect your database

In Supabase, create a new project. Once it's ready, go to Project Settings → API and copy two things: your Project URL and your anon public key.

Back in your project, create a file called .env in the root and paste them in. The VITE_ prefix is required.

.env
VITE_SUPABASE_URL=https://your-project.supabase.co
VITE_SUPABASE_ANON_KEY=your-anon-key-here

Now make one small file that connects to Supabase. Everything in the app will use this.

src/lib/supabase.js
import { createClient } from '@supabase/supabase-js'

const url = import.meta.env.VITE_SUPABASE_URL
const key = import.meta.env.VITE_SUPABASE_ANON_KEY

export const supabase = createClient(url, key)
✦ AI Prompt — copy & paste

"Create a Supabase client file at src/lib/supabase.js using @supabase/supabase-js. Read the URL and anon key from Vite env vars (VITE_SUPABASE_URL and VITE_SUPABASE_ANON_KEY) and throw a clear error if either is missing."

Step 5

Add login

A CRM needs a front door. Supabase gives you authentication for free. You'll build two pieces: an auth context that remembers who's logged in, and a login screen.

The auth context wraps your whole app and tracks the session. Here's the heart of it:

src/context/AuthContext.jsx (core)
// track the session and expose sign in / sign out
const [session, setSession] = useState(null)

useEffect(() => {
  supabase.auth.getSession().then(({ data }) => setSession(data.session))
  const { data } = supabase.auth.onAuthStateChange((_e, s) => setSession(s))
  return () => data.subscription.unsubscribe()
}, [])

const signIn = (email, password) =>
  supabase.auth.signInWithPassword({ email, password })

const signOut = () => supabase.auth.signOut()

Then your app shows the login screen when nobody's signed in, and the CRM when they are:

src/App.jsx (the gate)
if (loading) return <LoadingScreen />
if (!session) return <Login />
return <AppShell />   // the real CRM
✦ AI Prompt — copy & paste

"Build a React AuthContext using my Supabase client. It should track the session with getSession and onAuthStateChange, and expose signIn(email, password), signOut(), and the current user. Then create a dark, modern Login page with email + password fields that calls signIn and shows any error. Use Plus Jakarta Sans for headings and an orange gradient button."

Step 6

Build the shell — sidebar, header, routing

Every CRM has the same frame: a sidebar on the left to move around, a header on top, and a content area in the middle. Build it once and every page lives inside it.

Wire up routing in main.jsx with a BrowserRouter, then define your pages in App.jsx:

src/App.jsx (routes)
<Routes>
  <Route path="/" element={<Navigate to="/dashboard" />} />
  <Route path="/dashboard" element={<Dashboard />} />
  <Route path="/leads"     element={<Leads />} />
  <Route path="/contacts"  element={<Contacts />} />
  <Route path="/deals"     element={<Deals />} />
  <Route path="/settings"  element={<Settings />} />
</Routes>

The sidebar is just a list of links with icons. The active page gets a highlight. Don't hand-build all of it — describe it and let AI generate it.

Then build a sticky header with the page title, a search input, a notifications bell, and a red 'New' button."

Step 7

Build the dashboard

The dashboard is the home screen: a row of stat cards and a recent-activity feed. Each stat card is a small reusable component you feed a label, a number, an icon, and a trend.

src/components/StatCard.jsx (shape)
<StatCard label="Total Leads" value="1,284"
          icon={UserPlus} delta="+12.5%" />

Lay four of them in a grid, add a "Recent Activity" card underneath, and your home screen is done.

✦ AI Prompt — copy & paste

"Create a reusable StatCard component (label, value, icon, delta) styled as a dark rounded card with an icon chip and a trend pill. Then build a Dashboard page with a responsive grid of four StatCards (Total Leads, Active Deals, Revenue, Meetings This Week) and a 'Recent Activity' card below it that lists recent events with an icon, a description, and a timestamp."

Step 8

Add real data — your leads table

Now make it a real CRM. In Supabase, open the SQL Editor and create a leads table. This is where your actual contacts live.

Supabase SQL Editor
create table leads (
  id uuid primary key default gen_random_uuid(),
  name text not null,
  email text,
  company text,
  status text default 'new',
  value numeric default 0,
  created_at timestamptz default now()
);

Then read it into a Leads page. Supabase makes the query one line:

src/pages/Leads.jsx (fetch)
const { data: leads } = await supabase
  .from('leads')
  .select('*')
  .order('created_at', { ascending: false })

"Create a 'leads' table in Supabase with id, name, email, company, status, value, and created_at. Then build a Leads page in React that fetches all leads from Supabase, ordered by newest, and renders them in a dark table with a colored avatar, name + email, company, a status pill (new / qualified / negotiation / won), the value formatted as currency, and a relative 'last contact' time. Add a 'New Lead' button that opens a form to insert a row."

Repeat this same pattern for Contacts and Deals: a table in Supabase, a page that reads it, a form that writes to it. That's 80% of any CRM.

Step 9

Put it online

Your CRM runs on your machine. To use it anywhere, deploy it free on Vercel.

  • Push your project to a new GitHub repo.
  • Go to vercel.com, click Import Project, and pick your repo.
  • Add your two environment variables (VITE_SUPABASE_URL and VITE_SUPABASE_ANON_KEY) in the Vercel settings.
  • Click Deploy. In about a minute you'll have a live URL.

That's it. You now own a working CRM, running on the internet, that costs you nothing per month.
